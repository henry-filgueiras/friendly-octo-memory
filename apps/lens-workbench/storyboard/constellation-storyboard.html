Annotated storyboard

Workspace constellation turns run history into something you can read at a glance.

It is a playful view over the real `Artifact Lab` model: runs become stars, fork ancestry becomes arcs, artifact kinds become colors, and the current plus comparison sessions stay visibly marked.

Deterministic layout Fork ancestry Artifact-kind colors Current/comparison highlights Click to switch runs

How to use it

1. Create or import a few runs The view becomes interesting once you have at least one fork or comparison target.
2. Read the map Color means artifact kind, the warm halo is the current run, and the dashed ring marks the comparison run.
3. Click a node to jump there The constellation is an alternate session browser, not a separate model.

1. Build a little run universe

The normal run browser still tells you what exists.

Keep using checkpoints, forks, and imports the same way you already do. The constellation reads that durable workspace state; it does not invent a new branch model.

Same sessions, different lens The constellation reads the exact same run journals shown in the browser.
Forks matter Once you branch a run, the map has real ancestry to show instead of a flat list.

2. Read the constellation

The star map gives you a fast orientation layer.

Use it when you want the shape of the workspace more than the prose of the transcript: which run is current, which one you are comparing against, and how artifact heads differ by branch depth and kind.

Warm halo The current session stands out immediately so you always know which head you are driving.
Dashed ring The comparison session stays visible too, so the map reflects the same context as head comparison.
Curved ancestry The arcs are fork lineage, and the node color comes from the head artifact kind.
Workspace constellation is playful, but it stays grounded in the real append-only run model. It is a new way to read the workspace, not a second workflow engine.
